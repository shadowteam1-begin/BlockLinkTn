8
Deploy
1
Install Git and create GitHub account
Required by both Vercel and Railway
▼
  • 1
    Download Git from git-scm.com → install with default settings
  • 2
    Create a free account at github.com
  • 3
    Verify Git is installed — open terminal and run:
$ git --version # Should print: git version 2.x.x
2
Push your code to GitHub
One repo for frontend, one for backend
▼

Create two new repositories on GitHub (github.com → New repository). Make them public. Then run these commands:

Push the BACKEND:

# In your terminal, navigate to backend folder $ cd blood-alert-backend $ git init $ git add . $ git commit -m "BloodLink TN backend v3" $ git branch -M main $ git remote add origin https://github.com/YOUR_USERNAME/bloodlink-backend.git $ git push -u origin main

Push the FRONTEND:

$ cd ../blood-alert $ git init $ git add . $ git commit -m "BloodLink TN frontend v3" $ git branch -M main $ git remote add origin https://github.com/YOUR_USERNAME/bloodlink-frontend.git $ git push -u origin main
⚠
Your .gitignore already excludes .env and node_modules. Double-check before pushing — never upload secrets to GitHub.
3
Deploy backend to Railway
Free hosting for Node.js servers — railway.app
▼
  • 1
    Go to railway.app → Sign up with your GitHub account
  • 2
    Click "New Project" → "Deploy from GitHub repo"
  • 3
    Select your bloodlink-backend repository
  • 4
    Railway auto-detects Node.js and runs npm start
  • 5
    Click "Variables" tab → add ALL these environment variables:
# Paste each of these as KEY=VALUE in Railway Variables
MONGO_URI=mongodb+srv://blood_01:blood_2627@cluster0.9g8bd3g.mongodb.net/bloodlink?retryWrites=true&w=majority&appName=Cluster0
JWT_SECRET=bloodlink_tn_jwt_secret_2024_super_secure_key
JWT_EXPIRES_IN=7d
NODE_ENV=production
PORT=5000
FRONTEND_URL=https://your-app.vercel.app ← fill after Step 4
MSG91_AUTH_KEY=your_msg91_key_here
MSG91_TEMPLATE_ID=your_template_id
MSG91_SENDER_ID=BLDLNK
RAZORPAY_KEY_ID=rzp_test_xxxxxxxxxx
RAZORPAY_KEY_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxx
  • 6
    Click "Deploy" — Railway builds and starts your server
  • 7
    Go to Settings → Networking → Generate Domain to get your public URL

Your backend URL will look like:

LIVE https://bloodlink-tn-backend.up.railway.app

Test it — paste this in your browser:

https://bloodlink-tn-backend.up.railway.app/api # Should return JSON: { "success": true, "message": "🩸 BloodLink TN API v3.0" }
✓
Free Railway plan gives you $5 credit/month — enough to run your backend 24/7 for a college project.
4
Deploy frontend to Vercel
Free static hosting — vercel.com
▼
  • 1
    Go to vercel.com → Sign up with GitHub
  • 2
    Click "Add New Project" → Import your bloodlink-frontend repo
  • 3
    Framework Preset: Other (it's plain HTML, not React)
  • 4
    Root directory: leave as "/" (root)
  • 5
    Click "Deploy" — Vercel deploys in ~30 seconds

Your frontend URL will look like:

LIVE https://bloodlink-tn.vercel.app
ℹ
After getting your Vercel URL, go back to Railway → Variables → update FRONTEND_URL to your Vercel URL so CORS works correctly.
5
Update your frontend API URL
Point frontend to your live Railway backend
▼

Open blood-alert/js/api.js and update the PRODUCTION_URL with your actual Railway URL:

// Line 8 in api.js — change this to YOUR Railway URL const PRODUCTION_URL = 'https://bloodlink-tn-backend.up.railway.app';

Do the same in blood-alert/js/socket.js (line 27):

// Already auto-detects — just verify the URL matches const SOCKET_URL = (window.location.hostname === 'localhost' ...) ? 'http://localhost:5000' : 'https://bloodlink-tn-backend.up.railway.app';

Then commit and push the change:

$ cd blood-alert $ git add js/api.js js/socket.js $ git commit -m "Update production API URL" $ git push # Vercel auto-redeploys in ~20 seconds
6
Seed your live database
Run seed once against the deployed backend
▼

Your MongoDB Atlas URL is already in your .env. Run seed locally — it connects to the same cloud Atlas database your Railway backend uses:

$ cd blood-alert-backend $ npm run seed # Creates all test accounts, blood banks, sample data
✓
You only need to run this once. The data is stored in MongoDB Atlas which is shared between your local backend and Railway backend.

8
Auto-deploy on every code change
Push to GitHub → both sites update automatically
▼

Both Vercel and Railway watch your GitHub repos. Every time you push a change, they automatically rebuild and redeploy — no manual steps needed.

# Workflow after deploy: make a change, push, it's live $ git add . $ git commit -m "Fixed patient search bug" $ git push # → Vercel/Railway detects the push → rebuilds → live in ~45 seconds
