1
Create a free MongoDB Atlas account
Go to cloud.mongodb.com → sign up free
▼
  • 1
    Open your browser and go to cloud.mongodb.com
  • 2
    Click "Try Free" → Sign up with your email or Google account
  • 3
    Verify your email address (check inbox for the confirmation email)
  • 4
    Answer the setup questions: Choose "Learn MongoDB" or skip — doesn't matter
✓
MongoDB Atlas has a forever-free tier — 512MB storage, shared cluster. More than enough for your college project.
2
Create a free cluster
This is your database server in the cloud
▼
  • 1
    Click "Create" or "Build a Database" button
  • 2
    Choose "M0 Free" tier (the free option)
  • 3
    Cloud Provider: Choose AWS
  • 4
    Region: Choose "Mumbai (ap-south-1)" — closest to Tamil Nadu = faster
  • 5
    Cluster name: Leave as "Cluster0" or rename to "bloodlink"
  • 6
    Click "Create Cluster" — wait 1–3 minutes for it to set up

3
Create a database user
Username + password to access the DB from your backend
▼
  • 1
    On the left sidebar click "Database Access"
  • 2
    Click "Add New Database User"
  • 3
    Authentication Method: choose Password
  • 4
    Fill in a username and password — remember these!
  • 5
    Under "Built-in Role": select "Atlas Admin" or "Read and write to any database"
  • 6
    Click "Add User"
Example values (use your own!)
Username
bloodlinkadmin
Password
BloodLink@TN2024
⚠
Write down this username and password — you'll need them in Step 5 below to build the connection string. Do NOT use spaces or @ in the password.
4
Allow all IP addresses
So your laptop and deployment server can both connect
▼
  • 1
    On the left sidebar click "Network Access"
  • 2
    Click "Add IP Address"
  • 3
    Click "Allow Access From Anywhere" — this adds 0.0.0.0/0
  • 4
    Click "Confirm"
ℹ
0.0.0.0/0 means any IP can connect. This is fine for a college project. In production, you'd restrict this to your server's IP only.
5
Get the connection string
The URL your backend uses to talk to MongoDB
▼
  • 1
    Go to "Database" in the left sidebar
  • 2
    Click "Connect" next to your cluster
  • 3
    Choose "Drivers"
  • 4
    Driver: Node.js, Version: 5.5 or later
  • 5
    Copy the connection string shown — it looks like this:
mongodb+srv://bloodlinkadmin:<password>@cluster0.abcde.mongodb.net/bloodlink?retryWrites=true&w=majority
  • 6
    Replace <password> with the actual password you created in Step 3
  • 7
    Change myFirstDatabase (if shown) to bloodlink
6
Paste it into your .env file
Open blood-alert-backend/.env and update MONGO_URI
▼

Open blood-alert-backend/.env in VS Code and update it like this:

# MongoDB — paste your connection string here
MONGO_URI = mongodb+srv://bloodlinkadmin:BloodLink@TN2024@cluster0.abcde.mongodb.net/bloodlink?retryWrites=true&w=majority
# JWT Secret — change this to a long random string
JWT_SECRET = bloodlink_tn_super_secret_jwt_key_2024_changeme
PORT=5000
FRONTEND_URL=http://localhost:5500
🔒
Never share your .env file. Never upload it to GitHub. The .gitignore file in your backend already prevents this.
7
Test the connection & seed data
Verify it works, then fill the DB with sample data
▼

Open your terminal, navigate to the backend folder, and run these commands one by one:

1. Test MongoDB connection:

$npm run test:db
Expected output
🔍 Testing MongoDB connection...
✅ Connected to MongoDB Atlas!
📦 Database: bloodlink
🏠 Host: cluster0.abcde.mongodb.net
🎉 Your .env MONGO_URI is correct. Ready for Step 5!

2. Start the server:

$npm run dev

8
Connect frontend to backend
Replace mock data with real API calls using api.js
▼

Add api.js to your HTML pages, then replace the simulated timeouts with real calls. Example for the login form:

In pages/register.html — add this before </body>:

<script src="../js/api.js"></script>
<script src="../js/auth.js"></script>

In js/auth.js — replace the login setTimeout block:

// OLD — delete this block:
setTimeout(() => {
  setLoading('loginBtn', false);
  showFormAlert('loginAlert', 'success', '...');
}, 1400);

// NEW — paste this instead:
try {
  const data = await api.auth.login(email, password);
  window.location.href = getDashboardUrl(data.user.role);
} catch (err) {
  showFormAlert('loginAlert', 'error', err.message);
  setLoading('loginBtn', false);
}
✓
Every // BACKEND: comment in your dashboard JS files shows exactly where to make these replacements. Find them with Ctrl+F in VS Code.
